Repository Names

  • Prefix all whitelabel repository names:
    org_<domain/area>_*
  • Approved product domain prefixes are:
    • org_domaina_* (domain A)
    • org_domaina_* (domain B)
    • org_domainc_* (domain C)
    • org_domaind_* (domain d)
    • org_platform_* (platform domain, including DevOps/QA CoEs, and other cross-domain projects)
  • Approved internal domain prefixes are:
    • org_security_* (CISO and other global security projects)
    • org_internal_* (CIO and other internal backoffice projects)
    • org_hackathon_* (any hackathon repositories)
    • org_architecture_* (any global architecture projects, backend and frontend architecture guilds)
  • NOTE: Reach out to your domain/area leader if you want to introduce additional domain/area prefixes. Any changes should be approved in the CTO LT meeting where domain/area leaders meet weekly.
  • Use
    lowercase
    letters and digits
  • Separate words in the name with
    _
  • Avoid:
    • Team names in repository names
    • Personal names in repository names
